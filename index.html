解释

上述文本部分内容需要硬编码支持：

  • 1.首次解析，若存在 lists，则表示清单（比如所有进度的列表）
    若存在 CONTENT，表示其为需要的内容。 若存在 TYPE，表示其为内容的类型（应该默认识别为 JSON）。目前有以下几种：
    • [1]ARRAY 表示生成后应该为JSON 数组，例如： [{"a":"b"},{"a":"d"}]
    • [2]JSON 表示生成后为普通的 JSON，例如： {"a":"b","c":"d"}
  • 2. $类型名称|描述 表示该项目的类型以及描述（中间用“|”分割，描述中可能存在“|”）
  • 3. $text 类型为文本：String
  • 4. $double 类型为双精度小数（或者单精度小数）：double / float
  • 5. $long 类型为长整数（或者整数）：long long int / long int / int
  • 6. "T": "true" 此处意思为该类型（自定义类型）可以被单纯的文本替换
  • 7. {"DES": "描述", "TYPE": "JSON", "LISTCONTENT": { "key": { "amplifier": "$number|效果等级", "duration": "$number|效果持续时间，以刻为单位" } }这里定义了一个 JSON 控件。生成后的代码效果类似于（也就是说这个项目的名称需要被定义），类似于数组，但其项目可以增/删： {"minecraft:empty":{"amplifier":"1","duration":"2"}}
  • 8. 注意：如果一个项目的子项目包含“TYPE”应该优先处理“TYPE”标签，否则应该遍历子项目。可能为数组（未出现与内容中）
  • 9. "items": { "DES": "物品列表", "TYPE": "JSONARRAY", "LISTCONTENT": "$item|一个物品" } 类型为 JSONARRAY时候，应该读取 LISTCONTENT。此项目为 JSON 数组（生成后类似于： [{"a":"b"},{"a":"d"}] ） 一般而言此内容格式为：“$类型|描述”。但该项目也可以是一个 JSON，例如： "items": { "DES": "物品列表", "TYPE": "JSONARRAY", "LISTCONTENT": { "a":"$text|a", "b":"$text|b" } }
  • 10. "blocks": { "DES": "方块 ID 列表", "TYPE": "ARRAY" } 类型为 ARRAY 时，此项目为数组（每一项几乎都是文本）。
  • 11. 若存在 DES 项目，表示根项目的描述
  • 12. 若 CONTENT 中的子项目存在 LIST ，应该读取他，表示分类显示。例如： "generator": { "DES": "维度的生成设置", "type": { "TYPE":"text", "DES":"生成器的ID。前缀是minecraft:，之后是flat、noise或debug", "LIST":[ "minecraft:flat", "minecraft:noise", "minecraft:debug" ], "CONTENT":{ "minecraft:flat":{ ...(具体内容) } } } } 表示 generator下 type标签类型为 text，描述为 “生成器的ID。前缀是minecraft:，之后是flat、noise或debug”，支持的项目为 "minecraft:flat","minecraft:noise","minecraft:debug" （LIST中），每个项目对应的具体内容需要在 “CONTENT” 中对应的名称中。生成后的效果： {..."generator":{"type":"minecraft:flat",...(具体内容)}}
  • 13. $bool 逻辑型 bool
  • 14. $DF 密度函数。该值可以为一个密度函数或者一个整数。
  • 15. $json 表示 JSON文本（直接解析当成 JSON 拼接上去而不是当成 String）
